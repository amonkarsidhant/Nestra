# Nestra UX Flows (MVP)

## Design Principles
- Calm over clutter
- Explainability over magic
- Confirmation over accidental execution
- Trust signals over vanity metrics

## Flow 1: Ambient Home Entry
1. User lands on home screen
2. Sees household context, core status chips, and top outcomes
3. Can choose voice-first or quick action path

Success signal: user understands current state in under 10 seconds.

## Flow 2: Intent to Action (EV)
1. User picks low-tariff window
2. Confirms action
3. System returns status + policy reason + audit reference
4. KPI and audit refresh immediately

Success signal: user trusts result without opening another app.

## Flow 3: Scenario Outcome (Security/Comfort)
1. User taps scenario action
2. Busy state appears
3. Outcome rendered with clear status (accepted / pending / blocked)
4. Audit feed confirms execution trail

Success signal: user can verify what happened and why.

## Flow 4: Skeptical Buyer Check
1. User taps "Run skeptical buyer check"
2. UI computes live strengths and gaps from current signals
3. Verdict text openly compares Nestra and Home Assistant positioning

Success signal: transparency increases trust, even when gaps are acknowledged.

## Mobile / Tablet Rules
- Keep above-the-fold actions to 5-7 items maximum
- Primary action controls must remain thumb reachable
- Busy and error states must be explicit and non-blocking
- Preserve readability and spacing without dense dashboard patterns
